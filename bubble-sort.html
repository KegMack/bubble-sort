BUBBLE SORTING

This page will teach you about a specific algorithm used to sort arrays of data, called the "Bubble Sort." It is an iterative process in which a piece of data is compared to it's adjacent piece. If they are in the wrong order, they are swapped. This process is repeated until no more swaps are made, which indicates that the sorting is complete!

Sound complicated?

What just happened?

Here's how you do it, in leyman's terms: Take the last element of the list, and compare it to the one above it. If it's value is less, swap 'em! Go to the next to last element, and do the same thing. Repeat that until you get to the top of the list.

You can see that after the first pass, the element with the smallest value has 'bubbled' up to the top! So, we repeat the entire process, until no more swaps were made. Once that's happened, we are DONE!

How efficient is this sorting algorithm? Not very, when compared to other sorting methods. Check out the links below for info on other sorting methods. You may have noticed that the bubble sort iterates through all the elements on each pass, so for every additional element we sort, It can theoretically have to swap the entire list again. For this reason, the time it takes is a function of the square of the number of elements. This type of behavior is known in 'Big-O' notation as O(n2), where 'n' is the number of elements we need to sort.
